Rate-Based Cardiac Rhythm Triage

Sub-millisecond, sub-kilobyte rhythm-regime classifier. Triages beat-rate patterns (bradycardic, normal, tachycardic, transition-irregular) in 928 bytes of RAM using delay-line coincidence detection.

Validation status: first real MIT-BIH run available (v0.4, 5 records / 11 375 beats — 96.1 % pooled accuracy, 0.94 Irregular recall on AF/ectopic). See Real-data results below for the honest numbers and per-record breakdown. Synthetic P-QRS-T sweeps remain as the v0.2 hardening baseline.

Triage, not diagnosis. Flags beat-rate regimes only — not morphology-aware arrhythmia, not AF / VT / AVB / ST-elevation, not a substitute for clinical ECG. Targets the earlier layer in the monitoring chain.

Records 100 / 105 are mostly normal sinus — high accuracy (97 – 100 %). Records 200 (frequent PVCs) and 232 (atrial fibrillation) drop to ~91 % because rate-regime classification is fundamentally limited when ectopic beats or AF disrupt the RR pattern. AAMI columns are traceability only: CricketBrain does not classify N / S / V / F / Q morphology.

What this means: Normal sinus and Irregular (AF / ectopic) are detected with > 91 % recall. Tachycardia recall is only 57 % because most fast-RR windows in this set are part of AF and get correctly classified as Irregular. Bradycardia has zero ground-truth support in these 5 records — no claim about bradycardia performance can be made from this data. To evaluate Brady requires records 232 (covered) plus more slow-rhythm patients (e.g. 213, 232 sub-segments). Tracked as v0.4-followup.

CricketBrain's 96.1 % pooled accuracy on real MIT-BIH (5-record subset) is on rate-regime triage only; morphology-aware CNNs (Tiny MF-CNN, Stanford DNN) tackle the harder 5- and 12-class problems with 15 KB – 40 MB RAM and labelled training data. Different problems, different operating envelopes — see docs/competitive_analysis.md.
