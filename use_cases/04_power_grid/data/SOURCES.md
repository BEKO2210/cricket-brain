# Data Sources — Power-Grid Harmonic & Stability Triage

## Primary Dataset

**EPFL Smart-Grid Distribution Test Network**

The EPFL DESL Lab operates a real 20 kV smart-grid + PMU monitoring
infrastructure on the EPFL campus. The lab page documents historical
PMU data and a 20 ms refresh rate. **Verify every detail against the
official portal before relying on it.**

| Field | Value | Source |
|-------|-------|--------|
| Source | École Polytechnique Fédérale de Lausanne, DESL Lab | [DESL lab page](https://www.epfl.ch/labs/desl-pwrs/smart-grid/) |
| Network | Real 20 kV distribution test network on the EPFL campus | DESL portal |
| Refresh | 20 ms PMU reporting (50 frame/s synchrophasor) | DESL portal |
| Hardware tier | Synchrophasor units + auxiliary waveform capture | DESL portal |
| Format | CSV / HDF5 (typical for DESL releases) | research papers |
| License | CC BY 4.0 (typical for EPFL open data; **verify per release**) | EPFL Open Data policy |

**Important caveat.** The exact license, sampling rate of the auxiliary
waveform capture, and PMU hardware family of any specific data release
should be **verified directly on the EPFL DESL portal** at the URL
above before publication. Some details in earlier versions of this file
(e.g. "OpenPMU + EPFL synchrophasor units, 50 kHz aux waveform, CC BY 4.0")
were inferred from the lab's public materials but are not part of a
single canonical citation.

For the v0.1 release of this use case, no real EPFL data is bundled —
all benchmarks are on synthetic 50 Hz frequency-pattern streams
generated by `python/preprocess.py --synthetic`. Real-data validation
is the v0.2 priority milestone.

## Target Frequency Targets (50 Hz EU System)

| Channel | Frequency | Real-world meaning |
|---------|-----------|--------------------|
| FUND | 50 Hz | Healthy fundamental |
| H2 | 100 Hz | DC offset, transformer in-rush, half-wave rectified loads |
| H3 | 150 Hz | **Most common power-quality issue** — non-linear loads (rectifiers, VFDs, SMPS, LED ballasts, arc furnaces) |
| H4 | 200 Hz | Fast switching artefacts, EMI from RF power-electronics |

These four frequencies are exact integer multiples of 50 Hz so the
`TokenVocabulary::new(&[...], 50.0, 200.0)` distribution lands precisely
on each tuned channel.

## Usage Notes

- **CC BY 4.0** — attribution required, commercial use permitted.
- For 60 Hz systems (US, Canada, parts of South America, Japan partial)
  remap channels to 60 / 120 / 180 / 240 Hz by passing those values to
  `TokenVocabulary::new` in `src/detector.rs`.
- Off-nominal **fundamental drift** (49.5-50.5 Hz) is detected as
  `Nominal` by this triage detector. For ±0.1 Hz precision use a
  dedicated PMU and feed the precise frequency reading downstream.
